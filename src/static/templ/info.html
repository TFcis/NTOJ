NTOJ Guide

Verdicts

Verdict Full Name Conditions Explanation
AC Accepted Your program is correct!
It outputs the right answer in reasoneable time and within the limit memory usage.
你的程式輸出完全正確!
它在要求的時間和記憶體限制內輸出了正確答案。
WA Wrong Answer Your program didn't output the correct answer, but it terminated normally.
If you truly think your code is correct, contact the administrator through Question page.
你的程式輸出了錯誤的答案，但有正常結束。
如果你認為你的程式碼真的正確，可以透過Question頁面聯絡管理員。
TLE Time Limit Exceeded Your program tried to run for too much time.
If you are sure that the problem needs more time, please contact the administrator through Question page.
你的程式嘗試運行的時間太久。
如果你認為題目需要更長的時間限制，可以透過Question頁面聯絡管理員。
MLE Memory Limit Exceeded Your program tried to use more memory than the judge allows.
If you are sure that the problem needs more memory, please contact the administrator through Question page.
你的程式試圖使用的記憶體超出了題目的允許範圍。
如果你認為題目需要更大的記憶體範圍，可以透過Question頁面聯絡管理員。
RE Runtime Error Your program doesn't return a 0 code.
你的程式退出狀態不為 0。
RE Runtime Error(killed by signal) Your program failed during the execution.
There are some common reasons for this result:
  1. SIGSEGV - invalid memory access (segmentation fault)
  2. SIGFPE - erroneous arithmetic operation such as divide by zero
  3. SIGABRT - abnormal termination condition
    1. double free uninitialized or wrong memory address
    2. calling abort function
    3. assert
你的程式在執行過程中失敗了。
造成此結果有一些常見的原因:
  1. SIGSEGV - 無效記憶體存取 (記憶體區段錯誤)
  2. SIGFPE - 錯誤的算術運算，例如: 除以0
  3. SIGABRT - 異常終止條件
    1. 重複 free 未初始化或錯誤的記憶體位址
    2. 呼叫 abort() 函數
    3. assert
CE Compile Error The compiler fails to compile your program.
編譯器無法編譯你的程式。
IE Internal Error The judge is in a state of collapse.
Please contact the administrator through Question page.
評測系統處於崩潰狀態。
請透過Question頁面聯絡管理員。
PE Presentation Error Your output does not match with the test cases.
There may be extra blank lines and spaces.
你的輸出與測試案例不符。
可能會有多餘的空白行和空格。

Question Page

  • 提供使用者與管理員溝通的管道
  • 同時最多10筆發問，超過10筆會刪除最早的一筆發問

Challenges Filter

  • Problem ID: 題號間以 , 分隔, 例如: 1,2,3
  • Account ID: 使用者編號間以 , 隔開，例如: 1,2,3

Rank

  • rank 僅顯示每個user 最佳的submission
  • 先依照 Runtime 排序再依照 Memory 排序

Scoreboard

  • 記分板分數的顏色根據分數，以10分為一區間遞增
  • 滿分時(即AC)分數會以 粗體 顯示
  • 名次排序方式: 以分數為優先，若分數相同，以 AC submission count 較少者名次較前
  • 若想進入其他板塊，可透過Question頁面向管理員申請
